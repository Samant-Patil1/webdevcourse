Selectors & Specificity

Understanding CSS Selectors

CSS selectors are patterns used to select the elements you want to style. The selector tells the browser which HTML elements should be affected by the CSS declarations.

Specificity Hierarchy

When multiple rules apply to the same element, specificity determines which one wins:

  1. Inline styles - style="color: red" (1000 points)
  2. IDs - #header (100 points)
  3. Classes, attributes, pseudo-classes - .btn, [type], :hover (10 points each)
  4. Elements, pseudo-elements - p, ::before (1 point each)

Modern Selectors (CSS4+)

  • :is() - Groups selectors, matches specificity of most specific argument
  • :where() - Groups selectors with ZERO specificity
  • :has() - The "parent selector" — style elements based on their children!

Pro Tip: Avoid using !important. If you need it, your specificity is probably wrong. Use :where() for reset stylesheets to avoid specificity wars.

Box Model

The CSS Box Model

Every HTML element is a rectangular box. The box model describes how these boxes are sized and spaced:

  • Content - The actual content (text, images)
  • Padding - Space inside the border
  • Border - The edge of the box
  • Margin - Space outside the border

Box-Sizing Property

The default content-box adds padding and border to the element's total size. Use border-box to include padding and border in the width calculation:

/* Default (content-box) */
width: 300px + padding + border

/* border-box - RECOMMENDED */
width: 300px (includes padding + border)

Margin Collapsing

Vertical margins between block elements collapse (the larger value wins). This doesn't happen with horizontal margins or flex/grid items.

Modern Techniques

  • margin-inline: auto - Center block elements
  • margin-block - Vertical margins (writing-mode aware)
  • padding-inline, padding-block - Logical properties

Best Practice: Always set * { box-sizing: border-box; } in your CSS reset. It makes layouts predictable and intuitive.

Position Property

  • static - Default, normal document flow
  • relative - Offset from normal position, keeps space
  • absolute - Removed from flow, positioned to nearest positioned ancestor
  • fixed - Positioned to viewport, stays on scroll
  • sticky - Relative until scroll threshold, then fixed

Z-Index & Stacking Context

z-index only works on positioned elements (not static). Higher values appear on top. Creates new stacking contexts with:

  • position with z-index
  • opacity less than 1
  • transform, filter, clip-path
  • isolation: isolate

Centering Trick: Use position: absolute; inset: 0; margin: auto; for perfect centering without transforms!

Flexbox

What is Flexbox?

Flexbox is a one-dimensional layout method for arranging items in rows or columns. It's perfect for:

  • Navigation bars
  • Card layouts
  • Centering content
  • Distributing space

Container Properties

  • flex-direction - row (default) | row-reverse | column | column-reverse
  • flex-wrap - nowrap (default) | wrap | wrap-reverse
  • justify-content - Main axis alignment (flex-start, center, space-between, etc.)
  • align-items - Cross axis alignment (stretch, center, flex-start, etc.)
  • align-content - Multi-line spacing (when wrapped)
  • gap - Space between items (row-gap, column-gap)

Item Properties

  • flex-grow - How much item grows (0 = don't grow)
  • flex-shrink - How much item shrinks (1 = shrink)
  • flex-basis - Starting size before growing/shrinking
  • flex - Shorthand: grow shrink basis
  • align-self - Override container align-items for this item
  • order - Visual order (not DOM order)

Common Pattern: flex: 1 is shorthand for flex: 1 1 0% — item grows equally, shrinks equally, starts at 0.

CSS Grid

Flexbox vs Grid

FlexboxGrid
One-dimensionalTwo-dimensional
Content-firstLayout-first
Rows OR columnsRows AND columns
Great for componentsGreat for page layouts

Grid Template Functions

  • repeat() - repeat(3, 1fr) or repeat(auto-fit, 200px)
  • minmax() - minmax(200px, 1fr) min and max size
  • fit-content() - fit-content(300px) shrink-to-fit with max

Auto-Fit vs Auto-Fill

  • auto-fill - Creates empty tracks to fill space
  • auto-fit - Collapses empty tracks (stretches items)

Use auto-fit for most responsive layouts — it stretches items to fill space.

Grid Alignment

  • justify-items - Align items horizontally in their cell
  • align-items - Align items vertically in their cell
  • place-items - Shorthand for both
  • justify-content - Align entire grid horizontally
  • align-content - Align entire grid vertically

Responsive Grid Magic: grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); creates a responsive grid without any media queries!

Typography

Font Properties

  • font-family - Font stack: 'Inter', system-ui, sans-serif
  • font-size - Absolute (px), relative (em, rem), viewport (vw, vh)
  • font-weight - 100-900, or keywords (normal, bold)
  • font-style - normal, italic, oblique
  • line-height - Unitless preferred: 1.5 (multiplier)
  • letter-spacing - Space between letters
  • word-spacing - Space between words
  • text-transform - uppercase, lowercase, capitalize

Web Fonts

Load custom fonts from services like Google Fonts or self-host:

@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600&display=swap');

/* Or use link tag in HTML for better performance */
<link rel="preconnect" href="https://fonts.googleapis.com">

Modern Typography Features

  • font-display: swap - Show fallback font immediately, swap when loaded
  • clamp() - Fluid typography: font-size: clamp(1rem, 2.5vw, 2rem)
  • @font-face - Self-host fonts for better performance

Best Practice: Use rem for font sizes (relative to root), unitless line-height (1.5), and always include a system font fallback.

Why HSL is Best for Theming

HSL (Hue, Saturation, Lightness) is intuitive:

  • Hue (0-360) - The color (0=red, 120=green, 240=blue)
  • Saturation (0-100%) - Gray to full color
  • Lightness (0-100%) - Black to white

Want a lighter version? Just increase lightness! hsl(200 80% 50%) → hsl(200 80% 70%)

Gradient Types

  • linear-gradient - Directional fade between colors
  • radial-gradient - Circular/elliptical fade from center
  • conic-gradient - Colors around a center point (pie charts!)
  • repeating-* - Pattern gradients

Gradient Trick: Use transparent instead of rgba() for smoother edges: linear-gradient(to bottom, black, transparent)

Transforms & Transitions

2D Transforms

  • translate(x, y) - Move element
  • rotate(angle) - Rotate (degrees, radians, turns)
  • scale(x, y) - Resize (1 = normal, 2 = double)
  • skew(x, y) - Skew/shear
  • matrix() - Combine all (advanced)

3D Transforms

  • translate3d(x, y, z) - 3D movement
  • rotateX/Y/Z(angle) - Rotate around axis
  • scale3d(x, y, z) - 3D scaling
  • perspective - Create 3D depth

Transition Properties

  • transition-property - Which properties animate (all, none, or list)
  • transition-duration - How long (ms or s)
  • transition-timing-function - Speed curve
  • transition-delay - Wait before starting

Timing Functions

  • ease - Slow start, fast middle, slow end (default)
  • linear - Constant speed
  • ease-in - Slow start
  • ease-out - Slow end
  • cubic-bezier() - Custom curve
  • steps() - Step animation

Performance: Only animate transform and opacity for 60fps. They don't trigger layout or paint — just compositing!

Animations

@keyframes Rule

Define animations with @keyframes and apply with animation property:

@keyframes name {
  from { /* start state */ }
  to { /* end state */ }
  /* OR */
  0% { /* start */ }
  50% { /* middle */ }
  100% { /* end */ }
}

Animation Properties

PropertyDescription
animation-nameName of @keyframes
animation-durationHow long (ms or s)
animation-timing-functionSpeed curve
animation-delayWait before start
animation-iteration-countNumber or infinite
animation-directionnormal, reverse, alternate
animation-fill-modeKeep styles after animation
animation-play-staterunning or paused

Transitions vs Animations

  • Transitions - Triggered by state change (hover, click), two states only
  • Animations - Can run automatically, multiple keyframes, more control

Accessibility

Always respect prefers-reduced-motion:

@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}

Performance Tip: Use will-change: transform, opacity sparingly before animations, then remove it. Too many layers hurt performance!

Media Queries Mobile-First Breakpoints Container Queries

Responsive Design

Mobile-First Approach

Start with mobile styles, then enhance for larger screens:

/* Mobile styles (default) */
.card { width: 100%; }

/* Tablet and up */
@media (min-width: 768px) {
  .card { width: 50%; }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .card { width: 33.33%; }
}

Common Breakpoints

  • Mobile: 0 - 767px
  • Tablet: 768px - 1023px
  • Desktop: 1024px+
  • Large: 1440px+

Media Query Types

  • width/height - Viewport dimensions
  • orientation - portrait or landscape
  • prefers-color-scheme - dark or light mode
  • prefers-reduced-motion - accessibility
  • hover - Does device support hover?
  • pointer - coarse (touch) or fine (mouse)

Container Queries (Modern!)

Style based on container size, not viewport:

.card-container { container-type: inline-size; }

@container (min-width: 400px) {
  .card { display: flex; }
}

Fluid Design: Use clamp() for fluid typography and spacing. No more breakpoint hell!

CSS Variables & Themes

Custom Properties Basics

CSS Variables (Custom Properties) are defined with -- prefix and used with var():

:root {
  --primary: #3b82f6;
}

.button {
  background: var(--primary);
}

Variable Scope

  • :root - Global scope (available everywhere)
  • Any selector - Local scope (inherited by children)
  • Inline style - Element-specific

Dark/Light Theme Implementation

  1. Define semantic variables in :root (light theme)
  2. Override in [data-theme="dark"] selector
  3. Toggle with JavaScript by setting data-theme attribute
  4. Save preference to localStorage
  5. Check prefers-color-scheme for system preference

Design Token Strategy

Organize variables in layers:

  • Primitives: Raw values (--blue-500: #3b82f6)
  • Semantics: Purpose-based (--primary: var(--blue-500))
  • Components: Component-specific (--button-bg: var(--primary))

Pro Tip: Use HSL for color variables: --primary-h: 217; --primary: hsl(var(--primary-h) 90% 60%); Then adjust lightness for variants!

Best Practices & Build Challenge

BEM Naming Convention

Block Element Modifier — keeps CSS organized:

/* Block */
.card { }

/* Element (part of block) */
.card__title { }
.card__image { }
.card__button { }

/* Modifier (variation) */
.card--featured { }
.card--compact { }
.card__button--primary { }

CSS Organization Methods

  • ITCSS - Inverted Triangle: Settings → Tools → Generic → Elements → Objects → Components → Utilities
  • SMACSS - Base, Layout, Module, State, Theme
  • Atomic CSS - Utility-first (Tailwind approach)

Performance Best Practices

  • Avoid deep nesting (max 3 levels)
  • Don't over-qualify selectors (div.button → .button)
  • Minimize reflows (batch DOM changes)
  • Use transform and opacity for animations
  • Lazy load below-fold images
  • Compress and minify CSS

🎯 Build Challenge

Create a Responsive Pricing Page

Build a pricing page with these requirements:

  • 3 pricing cards in a responsive grid
  • Dark/light theme toggle (use CSS variables)
  • Hover animations on cards (transform + shadow)
  • Featured card stands out (bigger, different color)
  • Mobile: cards stack vertically
  • Tablet: 2 cards per row, 1 full width
  • Desktop: 3 cards side by side
  • Use BEM naming convention
  • Include CSS custom properties for theming

Remember: Write CSS that scales! Use variables, keep specificity low, and organize logically. Future you will thank present you.
