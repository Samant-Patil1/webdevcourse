Node.js Fundamentals

Node.js Basics

Node.js is a JavaScript runtime built on Chrome's V8 engine. It allows you to run JavaScript on the server side, opening up backend development to JavaScript developers.

Key Features

  • Non-blocking I/O - Handles many connections concurrently
  • Event-driven - Uses an event loop for async operations
  • Single-threaded - One thread handles all requests
  • NPM ecosystem - Access to millions of packages

Installation & Setup

  1. Download from nodejs.org
  2. Verify: node -v and npm -v
  3. Create a project: mkdir myapp && cd myapp
  4. Initialize: npm init -y

Global Objects

Unlike browsers that have window, Node has global. Key globals include __dirname, __filename, and process.

💡 Pro Tip: Use nodemon during development. It auto-restarts your server on file changes: npm install -g nodemon

File System Operations

File System

The fs module provides APIs to interact with the file system. Understanding synchronous vs asynchronous operations is crucial for building performant Node.js applications.

Sync vs Async

  • Synchronous - Blocks execution until complete. Use only at startup.
  • Asynchronous - Non-blocking, uses callbacks or promises. Always use in request handlers.

Common Operations

readFileRead entire file content
writeFileWrite/overwrite file
appendFileAdd content to end
unlinkDelete a file
mkdirCreate directory
readdirList directory contents

Streams

For large files, use streams to process data in chunks instead of loading everything into memory. This is memory-efficient and faster.

💡 Pro Tip: Always handle errors in file operations. Files might not exist, permissions might be denied, or disks might be full. Use try/catch with async/await for clean error handling.
03
HTTP Module & Server

HTTP Module

Node's http module lets you create web servers without external dependencies. While powerful, it's low-level compared to frameworks like Express.

Core Concepts

  • req (Request) - Contains client data: URL, headers, method, body
  • res (Response) - Used to send data back: status, headers, body
  • createServer() - Creates a server instance
  • listen() - Starts the server on a port

Common Status Codes

200OK - Success
201Created - Resource created
301/302Redirect
400Bad Request
401Unauthorized
404Not Found
500Server Error

Why Learn HTTP Module?

Understanding the HTTP module helps you appreciate what Express does for you. It's also useful for lightweight applications or when you want zero dependencies.

💡 Pro Tip: For real applications, use Express instead of raw HTTP module. Express provides cleaner routing, middleware support, and handles edge cases for you.
