HTML Introduction & Document Structure

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a web page using markup tags. HTML5 is the latest version, introducing semantic elements, multimedia support, and powerful APIs.

The Complete Document Structure

Every HTML document follows a specific structure. The <!DOCTYPE html> declaration defines this as an HTML5 document. The <html> element is the root, containing <head> (metadata) and <body> (visible content).

Essential Meta Tags

  • charset="UTF-8" - Character encoding for international characters
  • viewport - Responsive design control on mobile devices
  • description - Search engine snippet (150-160 chars ideal)
  • keywords - SEO keywords (less important now but good practice)

Open Graph Protocol

Open Graph meta tags control how your content appears when shared on social media (Facebook, LinkedIn, Twitter). Always include og:title, og:description, and og:image.

Place <script> tags just before the closing </body> tag to prevent blocking page rendering. Use defer or async attributes for scripts in the <head>.

Text Content & Typography

Heading Hierarchy (h1-h6)

Headings create a document outline. Only one h1 per page - it's the main topic. Search engines use headings to understand content structure. Screen reader users navigate by headings, making proper hierarchy crucial for accessibility.

Semantic Text Elements

  • <strong> - Important text (semantic)
  • <b> - Bold text (presentational, avoid)
  • <em> - Emphasized text (semantic)
  • <i> - Italic text (presentational, avoid)
  • <mark> - Highlighted/marked text
  • <small> - Side comments, copyright

Quotations

<blockquote> is for standalone quotes with optional cite attribute. <q> is for inline quotations - browsers automatically add quotation marks.

Computer Code

<code> for inline code, <pre> for preformatted blocks. Combine them for code blocks that preserve whitespace and use monospace font.

Element Display Usage
<h1>-<h6> Block Section headings
<p> Block Paragraphs
<br> Inline Line break (use sparingly)
<hr> Block Thematic break

Links & Navigation

URL Types

  • Absolute: Full URL with protocol (https://...)
  • Relative: Relative to current file (./page.html)
  • Root-relative: Starts from domain root (/contact)
  • Fragment: Links to page section (#section)

Security: rel="noopener noreferrer"

Always use when opening links in new tabs (target="_blank"). Prevents:

  • Tabnabbing: Malicious sites changing your page via window.opener
  • Privacy leak: Referrer information sent to external sites

Link Attributes

Attribute Purpose
href Link destination (required)
target _self, _blank, _parent, _top
rel Relationship (noopener, nofollow)
download Download file instead of navigate
hreflang Language of linked resource
type MIME type hint

Never use "Click here" as link text. Use descriptive text like "Read our pricing guide" for accessibility and SEO.

Images & Media

Critical Image Attributes

  • src: Image path (required)
  • alt: Alternative text for screen readers and when image fails to load
  • width/height: Prevent layout shift (CLS) by reserving space
  • loading: "lazy" defers off-screen images
  • decoding: "async" for better performance

Responsive Images (srcset)

srcset tells the browser which image to download based on viewport width and device pixel ratio (DPR). sizes tells the browser how wide the image will display at different breakpoints.

Art Direction with picture

Use <picture> when you need different image crops for different screen sizes (e.g., wide banner on desktop, square on mobile), not just different resolutions.

Video Attributes

Attribute Description
controls Show play/pause/volume controls
autoplay Auto-play (requires muted for most browsers)
muted Start muted
loop Loop playback
poster Thumbnail image before play
playsinline Play inline on mobile (not fullscreen)

Lists & Tables

List Types

  • <ul> - Unordered (bulleted) lists
  • <ol> - Ordered (numbered) lists with type, start, reversed
  • <dl> - Description lists for name-value pairs

Table Structure

  • <caption> - Table title/description
  • <thead> - Header section
  • <tbody> - Body/content section
  • <tfoot> - Footer (totals, summaries)
  • <th> - Header cells (use scope attribute!)
  • <td> - Data cells

Accessibility: scope Attribute

Always use scope="col" for column headers and scope="row" for row headers. This helps screen readers associate data cells with their headers.

Table Spans

  • colspan="n" - Cell spans n columns
  • rowspan="n" - Cell spans n rows

Don't use tables for layout! Use CSS Grid or Flexbox. Tables should only be used for tabular data.

Forms Part 1: Input Types

Form Element Attributes

  • action: URL where form data is sent
  • method: GET (URL params) or POST (body)
  • enctype: Encoding type (multipart/form-data for files)
  • autocomplete: Enable/disable browser autofill
  • novalidate: Disable HTML5 validation

Input Validation Attributes

Attribute Description
required Field must be filled
min/max Minimum/maximum value
minlength/maxlength Character limits
pattern Regex pattern validation
step Number increment step
placeholder Hint text

Always Associate Labels

Use <label for="id"> matching the input's id. This:

  • Expands clickable area (better UX)
  • Required for screen reader accessibility
  • Improves SEO

Use type="email", type="tel", type="url" instead of type="text" to get mobile-friendly keyboards and built-in validation.

Forms Part 2: Advanced Controls

Radio Buttons vs Checkboxes

  • Radio (<input type="radio">): Single choice from group. Same name attribute groups them.
  • Checkbox (<input type="checkbox">): Multiple selections allowed. Use value to identify which are checked.

Fieldset & Legend

<fieldset> groups related controls. <legend> provides a caption. Great for accessibility and visual grouping.

Select Element Variations

  • Basic: Single selection dropdown
  • Multiple: Hold Ctrl/Cmd to select multiple
  • Optgroups: Categorize options visually
  • Datalist: Provides autocomplete suggestions while allowing free text

Meter vs Progress

  • <progress> - Task completion (unknown max possible)
  • <meter> - Scalar measurement within a range (has min, max, low, high, optimum)

Button Types

Type Behavior
submit Submits the form (default)
reset Resets all form fields
button No default action (for JS)

Semantic HTML

Why Semantic HTML Matters

  • Accessibility: Screen readers announce element types
  • SEO: Search engines understand content structure
  • Maintainability: Code is self-documenting
  • Default Styling: Browsers apply appropriate defaults

Semantic Elements Guide

Element Use When
<header> Introductory content, navigation
<nav> Major navigation blocks
<main> Primary content (one per page)
<article> Self-contained content (blog post, news)
<section> Thematic grouping with heading
<aside> Tangential content (sidebar, pull quotes)
<footer> Footer content, copyright, links

Div vs Semantic Elements

Use <div> only when no semantic element fits. Ask: "What is this content's purpose?" before defaulting to div.

Always include a skip link (<a href="#main">) as the first focusable element for keyboard users to bypass navigation.

Interactive Elements HTML
Interactive Dialog Web Components

Interactive Elements

Details & Summary (Native Accordion)

No JavaScript required! <details> creates a disclosure widget. <summary> provides the always-visible heading. Use open attribute for default expanded state.

Dialog Element (Native Modal)

  • .show() - Opens as non-modal (non-blocking)
  • .showModal() - Opens as modal (blocks interaction, shows backdrop)
  • .close() - Closes the dialog
  • Form with method="dialog" auto-closes on submit

Iframe Security

Attribute Purpose
sandbox Restrict iframe capabilities
allow Permission policy (camera, mic, etc.)
referrerpolicy Control referrer information
loading Lazy loading support

Template Element

<template> holds HTML that is not rendered immediately. Use JavaScript to clone its .content and populate dynamic data. Perfect for list items, cards, or any repeating structures.

ARIA & Accessibility HTML

Accessibility & ARIA

ARIA Landmark Roles

When semantic HTML isn't possible, use ARIA roles to identify page regions: banner, navigation, main, complementary, contentinfo, search, form, region.

ARIA Labels

  • aria-label: Provides accessible name (invisible)
  • aria-labelledby: References visible text as label
  • aria-describedby: Additional description/context

ARIA States

Attribute Use Case
aria-expanded Dropdowns, accordions
aria-pressed Toggle buttons
aria-checked Custom checkboxes/radios
aria-selected Tabs, listbox options
aria-hidden Hide from screen readers

Live Regions

aria-live announces dynamic content changes:

  • off: Don't announce (default)
  • polite: Announce when user idle
  • assertive: Announce immediately

First rule of ARIA: If you can use native HTML elements, do that instead! ARIA doesn't change behavior, only announces information.

Graphics & Media APIs

Canvas vs SVG

Canvas SVG
Bitmap (raster) Vector
JavaScript-driven XML markup
Good for games, pixel manipulation Good for icons, charts, logos
Not accessible by default Accessible, searchable
Scales poorly Infinitely scalable

Canvas Context Methods

  • fillRect(x, y, w, h): Draw filled rectangle
  • strokeRect(x, y, w, h): Draw rectangle outline
  • clearRect(x, y, w, h): Clear rectangular area
  • arc(x, y, r, start, end): Draw circle/arc
  • fillText(text, x, y): Draw text

Web Storage API

  • localStorage: 5-10MB, persists indefinitely
  • sessionStorage: Same but cleared when tab closes
  • Both store strings only (use JSON.stringify/parse for objects)

Geolocation API

Requires HTTPS and user permission. Returns latitude, longitude, accuracy, altitude (if available). Always handle errors gracefully - user may deny permission.

HTML Best Practices & Challenge

HTML Style Guide

  • Always declare <!DOCTYPE html>
  • Use lowercase tag names
  • Close all elements (even optional ones like <p>)
  • Use lowercase attribute names
  • Always quote attribute values
  • Always specify alt, width, and height for images
  • Use lang attribute on html element
  • Place spaces around equals signs: name = "value" ❌ name="value" ✅

HTML Validation

Use the W3C Markup Validation Service to check your HTML for errors. Valid HTML ensures consistent rendering across browsers and better accessibility.

Character Encoding

Always use <meta charset="UTF-8"> as the first meta tag. This supports international characters (emojis, accented letters, non-Latin scripts).
