JS Basics

JavaScript Fundamentals

JavaScript is the programming language of the web. It runs in browsers and on servers (via Node.js), making it one of the most versatile languages.

Variable Declaration

Modern JavaScript uses let and const. Avoid var in new code.

  • let - Block-scoped, can be reassigned
  • const - Block-scoped, cannot be reassigned (but objects/arrays can be modified)

Data Types

JavaScript has 7 primitive types and objects. Everything that's not primitive is an object.

Strict Equality (===)

Always use === and !== instead of == and !=. Strict equality checks both value AND type, preventing unexpected type coercion.

💡 Pro Tip: Use const by default. Only use let when you know the value will change. This makes your code more predictable.
02
Control Flow

Control Flow

Control flow statements determine which code runs and when. Master these to create dynamic, responsive programs.

Conditional Statements

if...else chains and switch statements let your code make decisions based on conditions.

Loops

Choose the right loop for your task:

  • for - When you know the iteration count
  • while - When the condition is dynamic
  • for...of - For iterating over values (arrays, strings)
  • for...in - For iterating over keys (objects)

Loop Control

break exits the loop entirely. continue skips to the next iteration.

⚠️ Warning: Be careful with for...in on arrays - it iterates over ALL enumerable properties, not just indices. Use for...of or forEach for arrays instead.
03
Functions

Functions

Functions are the building blocks of JavaScript. They encapsulate reusable logic and enable powerful patterns like closures and higher-order functions.

Function Types

  • Declaration - Hoisted, named function
  • Expression - Assigned to a variable
  • Arrow - Concise syntax, lexical this

Scope & Closures

JavaScript has function scope and block scope (with let/const). A closure is a function that "remembers" its outer scope even when called elsewhere.

Arrow Functions vs Regular

Arrow functions don't have their own this, arguments, or super. They're perfect for callbacks and when you don't need function-specific features.

💡 Best Practice: Use arrow functions for short callbacks and when you want to preserve this from the outer scope. Use regular functions for methods and when you need the function's own this.

Arrays & Objects

Arrays & Objects

Arrays and objects are JavaScript's primary data structures. Master their methods and manipulation techniques to handle complex data efficiently.

Array Methods

Modern JavaScript provides powerful array methods:

  • Transform - map, flatMap
  • Filter - filter, find, findIndex
  • Aggregate - reduce, some, every
  • Sort - sort (mutable), toSorted (immutable)

Destructuring

Extract values from arrays and objects into variables cleanly.

Spread & Rest

... spreads elements (copy/concat) or collects them (rest parameters).

💡 Remember: Arrays are objects in JavaScript. Use Array.isArray() to check if something is specifically an array.

DOM Manipulation

DOM Manipulation

The Document Object Model (DOM) is JavaScript's interface to HTML. Master DOM manipulation to create dynamic, interactive web pages.

Element Selection

  • querySelector - Single element (CSS selector)
  • querySelectorAll - All matching elements
  • getElementById - Fastest for ID lookups

NodeList vs HTMLCollection

querySelectorAll returns a static NodeList. getElementsBy* returns a live HTMLCollection that updates when DOM changes.

Performance Tips

Cache selectors in variables. Use textContent over innerHTML for plain text. Batch DOM changes to minimize reflows.

⚠️ Security: Never use innerHTML with user input - it opens XSS vulnerabilities. Use textContent instead.
06
Events

Events

Events enable interactivity. They signal when something happens - a click, keypress, page load, or custom trigger.

Event Handling Methods

  • addEventListener - Modern, multiple handlers per event
  • onclick (inline) - Old way, overwrites previous handlers

Event Propagation

Events bubble up from target to ancestors (bubbling phase). Capturing goes the opposite direction. Most handlers use bubbling.

Event Delegation

Instead of adding listeners to many elements, add one to a parent and check event.target. More efficient for dynamic content.

💡 Performance: Always remove event listeners when components are destroyed to prevent memory leaks, especially in SPAs.

Forms & Input

Forms & Input

Forms are the primary way users interact with websites. JavaScript enhances forms with validation, dynamic updates, and AJAX submission.

FormData API

The FormData object makes it easy to collect and send form values. It handles all input types including files.

Validation API

HTML5 validation attributes (required, pattern, etc.) work with JavaScript's validity API for custom error handling.

File Handling

Use FileReader to read file contents. URL.createObjectURL() creates temporary URLs for previews.

💡 UX Tip: Always validate on both client-side (for speed) AND server-side (for security). Client validation can be bypassed!

JSON & Data

JSON & Data

JSON (JavaScript Object Notation) is the universal data format for APIs. It's lightweight, human-readable, and supported by every modern language.

JSON.stringify

Converts JavaScript objects to JSON strings. Use replacer to filter properties, space for pretty printing.

JSON.parse

Converts JSON strings back to JavaScript objects. Always wrap in try/catch when parsing external data.

Working with APIs

APIs return JSON. Use Response.json() to parse, or JSON.parse() for raw strings.

⚠️ Limitations: JSON doesn't support functions, undefined, or circular references. Dates become strings - use libraries like date-fns for proper handling.

Fetch API

Fetch API

The Fetch API is the modern way to make HTTP requests in JavaScript. It returns Promises and provides a cleaner interface than the older XMLHttpRequest.

Key Concepts

  • fetch() returns a Promise that resolves to a Response
  • response.ok is true for status 200-299
  • response.json() parses JSON response
  • AbortController cancels in-flight requests

Always Check response.ok

Fetch doesn't reject on HTTP error codes (4xx, 5xx). You must check ok or status manually.

💡 Best Practice: Use async/await for cleaner code, and always wrap fetches in try/catch. Create a wrapper function for consistent error handling across your app.

Async Programming

Async Programming

JavaScript is single-threaded, but async operations let you handle tasks like API calls without freezing the UI. Understanding async patterns is essential for modern web development.

Evolution of Async

  1. Callbacks - Original approach, leads to "callback hell"
  2. Promises - Cleaner, chainable async handling
  3. Async/Await - Syntactic sugar over Promises, most readable

Promise States

  • pending - Initial state
  • fulfilled - Operation completed successfully
  • rejected - Operation failed
💡 Remember: async/await is just Promises with better syntax. Every async function returns a Promise, and await can only be used inside async functions.

Storage

Storage & Persistence

Web storage APIs let you save data in the browser. Choose the right storage based on your needs for persistence, size, and security.

Storage Options

Type Persistence Size Sent to Server
localStorage Until deleted ~5-10MB No
sessionStorage Tab session ~5-10MB No
Cookies Configurable ~4KB Yes
IndexedDB Until deleted Large No

Security Considerations

  • Never store sensitive data (passwords, tokens) in localStorage
  • Use secure and HttpOnly cookies for auth
  • Always validate data retrieved from storage
⚠️ XSS Warning: localStorage is accessible to any JavaScript on your domain. If an XSS vulnerability exists, attackers can steal stored data.

Modern JS

Modern JavaScript (ES6+)

ES6 (2015) and subsequent versions added powerful features that make JavaScript more expressive and easier to write. Master these for cleaner, more maintainable code.

Key ES6+ Features

  • Template Literals - Backtick strings with ${interpolation}
  • Destructuring - Extract values from arrays/objects
  • Spread/Rest - ... operator for expansion/collection
  • Arrow Functions - Concise function syntax
  • Modules - import/export for code organization
  • Optional Chaining - ?. for safe property access
  • Nullish Coalescing - ?? for default values
💡 Compatibility: All modern browsers support ES6+. For older browsers, use a transpiler like Babel.

Error Handling

Error Handling

Robust applications handle errors gracefully. JavaScript provides several mechanisms for catching and managing errors to prevent crashes and provide good user experiences.

Error Handling Patterns

  • Try/Catch - Wrap risky synchronous code
  • Async Try/Catch - Handle promise rejections
  • Global Handlers - Catch unhandled errors
  • Custom Errors - Create meaningful error types

Common Errors

  • TypeError - Wrong type operation
  • ReferenceError - Undefined variable
  • SyntaxError - Invalid code
  • RangeError - Number out of range
💡 Debugging Tip: Use console.table() for arrays/objects, console.trace() to see the call stack, and the debugger statement to pause execution at specific lines.

Best Practices & Challenge

Writing clean, maintainable JavaScript is a skill that comes with practice. Follow these best practices and then test your skills with the Notes App challenge.

Code Quality Guidelines

  • Use meaningful variable names (isLoading vs flag)
  • Keep functions small and focused (single responsibility)
  • Avoid global variables (use modules or closures)
  • Use const by default, let when needed
  • Always handle errors (don't ignore catch blocks)
  • Comment the "why", not the "what"

Performance Tips

  • Cache DOM queries in variables
  • Debounce input handlers
  • Use event delegation for dynamic lists
  • Lazy load non-critical resources
